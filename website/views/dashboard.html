Help - Github Push Events

What does this show me?

We’ve taken all your pushes to the public repositories and visualized your weekly push patterns. We’ve broken the week down into hourly buckets and counted how many pushes you have done in each hour. The more pushes, the darker the square in the heatmap.

Try squinting at the picture - what does the overall pattern tell you? Try scanning horizontally along the days to see the daily pattern of when you push. Which days do you push most? Do you push on the weekend? Try scanning vertically, is there a pattern? Do you push before lunch? Before you finish working for the day? Are there parts of the week that don’t have any pushes at all? Why is that?

How do I get data into this graph?

You need to have pushed to a public repository on GitHub to see something in this graph. Then, just click on ‘Link Data’, “Link Up My Public GitHub Data’ to get started.

When will I see something interesting?

Immediately! GitHub keeps all your pushes in a useful historical data source. This means that we are able to retrieve data reaching back before you signed up to QuantifiedDev.

How do I update the data?

Right now you have to do that manually - what a drag! Don’t worry, it’s easy. Just hit the refresh button on the header of the graph, it’s the one with two arrows in a circle. When you hit refresh, we go back to github and sync up the data.

Can I see pushes to private repos?

Not yet, it’s coming soon though. You can bump it up the queue by letting us know this is important to you, just go to http://quantifieddev.uservoice.com and send us a request.

Help - Active Programming Duration

What does this show me?

This graph shows you how much time you spent each day actively creating code. Our plugins track your mouse clicks and edits to determine how long you are reading or writing code. If you spend a long time thinking, or debugging, or waiting for builds, none of that time is included. The only time included is when you are browsing your source files or editing them. We take all the activity over each day for the last month, add it up, then show it to you. The data goes back over a month.

How do I get data in this graph?

To see data in this graph you have to install one of the plugins for the IDEs that we support. Currently that’s Sublime Text, IntelliJ or Visual Studio. Go to the downloads page and follow the instruction there to get started.

When will I see something interesting?

You’ll start seeing data as soon as you have had your first programming session with the plugin installed. A programming session is created when you are active in your IDE and then head off to a meeting, for lunch or to check hackernews, slashdot or twitter. Do you know how much time you spend actively developing? After just a single day, you’ll know something new about your development self! Make sure you check back on the dashboard to see the patterns emerge as you build your data up towards a full month.

Can I collect data from multiple IDEs? From work? Home?

Absolutely! Our goal at QuantifedDev is to give you a complete picture of your development life, that includes all of the IDEs and all of the computers that you use. Simply install our plugins on all your IDEs, across all of your computers.

My IDE isn’t support, what do I do?

We’d love you to get involved and build a plugin for it! We’ve created QuantifiedDev as open source and not for profit so that you can get involved. Our api is simple and we can give you the algorithms we use in the other plugins to calculate activity. Email team@quantifieddev.org if you want to help your community and enable QuantifiedDev support in your IDE. You can also make a request on quantifieddev.uservoice.com. The more people who want the plugin, the more likely we are to build it. So get your community behind the request by upvoting it.

How do I update the data?

Kick back on your sofa and relax. It’s automatic. The plugins collect and send the data as you spend time programming in your IDE. As soon as the data arrives at our server, it’s available in your visualisation.

See something interesting?

Show us! We’d love hearing the stories behind the data. Send your story to team@quantifieddev.org

Help - Correlate IDE Activity with GitHub pushes

What does this show me?

If you have both IDE activity data and GitHub pushes, you will see a correlation between the number of minutes spent active in your IDE and the number of pushes to your public repositories. To generate the correlation we create totals for IDE activity and GitHub pushes for each day. Then, we plot a point for each day. We use the x-axis for IDE activity and the y-axis for GitHub pushes.

How do I get data into this graph?

To see correlations you must link up your GitHub data and install a plugin that collects IDE activity data.

IDE Activity

Currently that’s Sublime Text or IntelliJ. Go to the downloads page and follow the instructions there to get started.

GitHub pushes

For the GitHub pushes, you need to have pushed to a public repository on GitHub to see something in this graph. If you have, go to the dashboard, click on ‘Link Data’, “Link Up My Public GitHub Data’ to get started.

When will I see something interesting?

GitHub is a historical data source, which means we get lots of juicy data the second you connect it up. IDE activity takes a bit longer as we collect that as you complete programming sessions. To see something interesting you need to wait until you see lots of days where you were active in your IDE and also pushing up to GitHub. It will take a few weeks before you see this. We recommend leaving the data collection going and then coming back after 2 weeks. Why not set

Not yet, it’s coming soon though. You can bump it up the queue by letting us know this is important to you.

How do I update the data?

Right now, you need to sync the GitHub push data manually. IDE activity on the other hand updates automatically. To update the GitHub push data, simply go to the GitHub Push Events graph and click no the refresh button. It’s in the graph header, it’s the one with two arrows in a circle.

Help - Hourly WTFs

What does this show me?

You know those moments where you’re reading code - probably your own - it doesn’t make sense and as you realise what the code is doing you think: WTF?!

This graph shows you all the moments you slapped your head and thought “WTF?!”. Are you grumpier in the morning, therefore logging more WTFs? Maybe you’re sharper before lunch, code is less confusing: fewer WTFs.

How do I get data into this graph?

This data can only be logged from our IntelliJ plugin. Once installed, whenever you have a WTF, simply hit the WTF button. Then it will appear immediately in the visualisation.

When will I see something interesting?

Depends on how bad your code base is ;-)

As soon as you hit the WTF button you’ll see the data appear in the visualisation. Check the dashboard every friday to see your week in WTFs. Use WTFs in your retrospective to see whether your code quality is going up or down.

My IDE isn’t support, what do I do?

We’d love you to get involved and build a plugin for it! We’ve created QuantifiedDev as open source and not for profit so that you can get involved. Our api is simple and it’s really easy to create a button that will log a WTF. Email team@quantifieddev.org if you want to help your community and enable QuantifiedDev support in your IDE. You can also make a request on quantifieddev.uservoice.com. The more people who want the plugin, the more likely we are to build it. So get your community behind the request by upvoting it..
