Help - Github Push Events

What does this show me?

We’ve taken all your pushes to the public repositories and visualisated your weekly push patterns. We’ve broken the week down into hourly buckets and counted how many pushes you have done in each hour. You should see something about your weekly habits.

Try scanning horizontally along the days to see the daily pattern of when you push. Which days do you push most? Do you push on the weekend? Try scanning vertically, is there a pattern? Do you push before lunch? Before you finish working for the day? Are there parts of the week that don’t have any pushes at all? Why is that?

Linking your data

You need to have pushed to a public repository on GitHub to see something in this graph. Then, just click on ‘Link Data’, “Link Up My Public GitHub Data’ to get started.

How quickly will I see my data?

Straight away! GitHub holds this historical data which means that we show your patterns immediately.

Can I see pushes to private repos?

Not yet, it’s coming soon though. You can bump it up the queue by letting us know this is important to you.

See something interesting?

Show us! We’d love hearing the stories behind the data. Send your story to team@quantifieddev.org

Help - Hourly Builds

What does this show me?

If you are working in IntellJ or Visual Studio, a build event is created each time you start a build. The hourly builds shows the pattern of when you were building over each day of the week. What patterns do you see? Do you break for lunch at the same time every day? Do you program on the weekends? Are you a morning person or a night owl?

Closely tied to this visualisation is the build history and build duration visualisations. Once you start collecting builds, these visualisation will automatically appear in your dashboard.

Getting the data

To see data in this graph you have to install one of the plugins for the IDEs that we support: Sublime Text, IntelliJ or Visual Studio. This data is collected from the point at which you install the plugin, so get started now by going to the downloads page.

You can install plugins by going to the downloads page and following the instructions for your IDE. Don’t forget to register the IDE plugin by following the link to QuantifiedDev in the plugin.

If you use multiple IDEs or multiple computers, install the plugin on each IDE that you use.

How quickly will I see my data?

Straight away! GitHub holds this historical data which means that we show your patterns immediately.

Help - Active Programming Duration

What does this show me?

This graph shows you how much time you spent each day actively creating code. Our plugins track your mouse clicks and edits to determine how long you are reading or writing code. If you spend a long time thinking, or debugging, or waiting for builds, none of that time is included. The only time included is when you are browsing your source files or editing them.

Getting the data

To see data in this graph you have to install one of the plugins for the IDEs that we support: Sublime Text, IntelliJ or Visual Studio. This data is collected from the point at which you install the plugin, so get started now by going to the downloads page.

How quickly will I see my data?

Straight away, however you will only see your data from the point of installation of our plugin. The best thing to do is to install the plugin and revisit the dashboard in a couple of days.

Can I see pushes to private repos?

Not yet, it’s coming soon though. You can bump it up the queue by letting us know this is important to you.

Help - Correlate IDE Activity with GitHub pushes

What does this show me?

If you have both IDE activity data and GitHub pushes, you will see a correlation between the number of minutes spent active in your IDE and the number of pushes to your public repositories. There is a data point for each day that there is either IDE acitivty or github pushes.

Getting the data

To see correlations you must link up your GitHub data and install at least one plugin for your IDE(s).

For the GitHub pushes, you need to have pushed to a public repository on GitHub to see something in this graph. If you have, just click on ‘Link Data’, “Link Up My Public GitHub Data’ to get started.

To see data in the IDE activity data install one of the plugins for the IDEs that we support: Sublime Text, IntelliJ or Visual Studio. This data is collected from the point at which you install the plugin.

How quickly will I see my data?

Since the IDE activity is collected from the point of install, correlations will appear for days from the point of installation of the plugin. Check back regularly to see the correlations appear.

Can I see pushes to private repos?

Not yet, it’s coming soon though. You can bump it up the queue by letting us know this is important to you.
